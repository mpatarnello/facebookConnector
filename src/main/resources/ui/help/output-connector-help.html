Twitter is... Twitter. It needs no introduction. See online docs for more details (and demos).

Note: Integrations using the Twitter components can only be run on FoxWeave infrastructure i.e. you cannot use them On-Premise or On-PaaS. This is because of the type of authentication used by the Twitter service (OAuth v1.0a), which requires the use of a "consumerSecret", which must be protected and so cannot be exposed to non-FoxWeave infrastructure (i.e. On-Premise or On-PaaS machines).

Configuring the Twitter output connector is quite simple:

  1. Select the Twitter Account (press the button to add an account, if needed).
  2. Select the Twitter Action that you wish to sync/migrate to the selected account.
  3. Go to the Data Mapping tab and map the pipeline data into the Zendesk object (selected in step #2).
